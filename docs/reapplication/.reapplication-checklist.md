# Reapplication Checklist

Use this checklist to track your progress through the reapplication process.

## Pre-Reapplication

- [ ] Read `README_REAPPLICATION.md`
- [ ] Understand what changed (see `SESSION_CHANGES_SUMMARY.md`)
- [ ] Have `CHANGES_TO_REAPPLY.md` open for reference
- [ ] Backup any uncommitted work: `git stash`

## Git Merge

- [ ] Switch to main: `git checkout main`
- [ ] Pull latest: `git pull origin main`
- [ ] Switch to punyak: `git checkout punyak`
- [ ] Merge main: `git merge main`
- [ ] Resolve conflicts (if any)
- [ ] Commit merge: `git add . && git commit -m "Merge main into punyak"`

## Reapplication

### Option A: Automated (Recommended)
- [ ] Run script: `bash reapply-session-changes.sh`
- [ ] Follow prompts
- [ ] Review changes

### Option B: Manual
- [ ] Create `backend/api/routes/__init__.py`
- [ ] Create `backend/api/routes/quiz.py` (copy from CHANGES_TO_REAPPLY.md)
- [ ] Create `backend/api/routes/profile.py` (copy from CHANGES_TO_REAPPLY.md)
- [ ] Fix `frontend/src/services/vibeGraphAPI.js` (API URL)
- [ ] Update `docker-compose.yml` (DynamoDB, LocalStack, healthcheck)
- [ ] Update `docker-compose.override.yml` (volumes, environment)
- [ ] Move `backend/utils/connection_check.py` to `backend/src/utils/`
- [ ] Create `backend/src/utils/__init__.py`
- [ ] Update `backend/api/startup.py` (import path)
- [ ] Update `backend/handlers/Dockerfile` (CMD)
- [ ] Update `backend/services/Dockerfile` (CMD)

## Verification

### File Checks
- [ ] `backend/api/routes/quiz.py` exists
- [ ] `backend/api/routes/profile.py` exists
- [ ] `backend/src/utils/connection_check.py` exists
- [ ] `frontend/src/services/vibeGraphAPI.js` has `VITE_API_URL`
- [ ] `backend/api/startup.py` imports from `src.utils`

### Build and Start
- [ ] Clean: `make clean`
- [ ] Build: `make build` (no errors)
- [ ] Start: `make up` (all containers start)
- [ ] Wait: `make wait-healthy` (all healthy)

### Container Health
- [ ] `vibegraph-frontend` - healthy
- [ ] `vibegraph-backend-api` - healthy
- [ ] `vibegraph-backend-handlers` - running
- [ ] `vibegraph-backend-services` - running
- [ ] `vibegraph-dynamodb-local` - healthy
- [ ] `vibegraph-localstack` - healthy

### DynamoDB Tables
- [ ] `vibegraph-users` exists
- [ ] `vibegraph-sessions` exists
- [ ] `vibegraph-embedding-cache` exists

### API Endpoints
- [ ] Health: `curl http://localhost:8000/health` returns `{"status":"healthy"}`
- [ ] Ready: `curl http://localhost:8000/health/ready` returns ready status
- [ ] Quiz: `curl -X POST http://localhost:8000/quiz/section1/start -H "Content-Type: application/json" -d '{}'` returns sessionId
- [ ] Profile: `curl http://localhost:8000/profile/dna/test-user` returns taste DNA

### Frontend
- [ ] Frontend loads: `curl http://localhost:3000` returns HTML
- [ ] Page title: HTML contains "VibeGraph"

- [ ] Can start quiz in browser

### Integration
- [ ] Frontend can call backend APIs
- [ ] Quiz flow works end-to-end
- [ ] No network errors
- [ ] No authentication errors (expected for now)

## Post-Reapplication

- [ ] All tests pass
- [ ] No errors in logs

- [ ] Commit changes: `git add . && git commit -m "Reapplied session changes"`
- [ ] Push: `git push origin punyak`

## Troubleshooting (if needed)

If any check fails:
- [ ] Check logs: `docker logs <container-name>`
- [ ] Review `CHANGES_TO_REAPPLY.md` for exact changes
- [ ] Check `QUICK_REAPPLY_COMMANDS.md` for troubleshooting commands
- [ ] Try clean rebuild: `make clean && make build && make up`
