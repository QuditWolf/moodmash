# Backend Documentation

## Overview

The VibeGraph backend is a serverless architecture built on AWS Lambda functions that provides adaptive quiz generation, taste profile embedding, DNA archetype creation, growth path recommendations, taste matching, and behavioral analytics. The system leverages AWS Bedrock (Claude 3.5 Sonnet and Titan v2) for AI processing while maintaining strict privacy-first principles by storing only embeddings rather than raw quiz responses.

## Architecture

The backend is organized into three main layers:

### 1. Handlers Layer (`src/handlers/`)
Lambda functions that serve as entry points for API requests. Each handler processes HTTP requests, validates inputs, orchestrates service calls, and returns formatted responses.

**Available Handlers:**
- [generateSection1](./handlers/generateSection1.md) - Generate foundational quiz questions
- [generateSection2](./handlers/generateSection2.md) - Generate adaptive follow-up questions
- [generateEmbedding](./handlers/generateEmbedding.md) - Create 1024-dim taste embeddings
- [generateDNA](./handlers/generateDNA.md) - Generate taste DNA archetypes
- [generatePath](./handlers/generatePath.md) - Create personalized growth paths
- [findMatches](./handlers/findMatches.md) - Find taste-based user matches
- [generateAnalytics](./handlers/generateAnalytics.md) - Generate behavioral insights
- [healthCheck](./handlers/healthCheck.md) - System health monitoring

### 2. Services Layer (`src/services/`)
Reusable service modules that encapsulate business logic and external integrations. Services handle AWS SDK interactions, caching, and data persistence.

**Available Services:**
- [bedrockClient](./services/bedrockClient.md) - AWS Bedrock API client
- [claudeService](./services/claudeService.md) - Claude 3.5 Sonnet integration
- [titanEmbeddingService](./services/titanEmbeddingService.md) - Titan v2 embedding generation
- [dynamoClient](./services/dynamoClient.md) - DynamoDB operations
- [cacheService](./services/cacheService.md) - Embedding cache management
- [userService](./services/userService.md) - User data operations

### 3. Utils Layer (`src/utils/`)
Utility functions for common operations like validation, hashing, logging, and vector mathematics.

**Available Utils:**
- [validator](./utils/validator.md) - Input validation utilities
- [hash](./utils/hash.md) - SHA-256 hashing for cache keys
- [logger](./utils/logger.md) - Structured logging utilities

### 4. Specialized Modules

#### Embedding Module (`src/embedding/`)
- [embeddingBuilder](./utils/embeddingBuilder.md) - Build structured embedding documents
- [normalizeVector](./utils/normalizeVector.md) - Vector normalization
- [weightingEngine](./utils/weightingEngine.md) - Apply weighting to embeddings

#### Matching Module (`src/matching/`)
- [cosineSimilarity](./utils/cosineSimilarity.md) - Calculate vector similarity
- [matchEngine](./utils/matchEngine.md) - Taste matching logic

## Data Flow

```
User Request → API Gateway → Lambda Handler → Services → AWS Bedrock/DynamoDB
                                    ↓
                              Response ← Format ← Process ← Results
```

## Key Features

### Privacy-First Design
- Raw quiz answers are NEVER stored in the database
- Only 1024-dimensional embedding vectors are persisted
- Embeddings are mathematically irreversible to original answers

### Intelligent Caching
- Embedding cache uses SHA-256 hashing of embedding documents
- Cache hit rate target: 40%
- Reduces Titan API calls and costs

### Adaptive Quiz Generation
- Section 1: 5 foundational questions generated by Claude
- Section 2: 5 adaptive questions based on Section 1 responses
- Questions tailored to user's taste patterns

### Vector Operations
- 1024-dimensional embeddings from Titan v2
- Cosine similarity for taste matching (threshold: 0.7)
- Vector normalization to unit length

## Environment Configuration

Required environment variables:

```bash
# AWS Configuration
AWS_REGION=us-east-1
DYNAMODB_ENDPOINT=http://localhost:8000  # For local development
BEDROCK_ENDPOINT=http://localhost:4566   # For LocalStack

# Table Names
USERS_TABLE=vibegraph-users
SESSIONS_TABLE=vibegraph-sessions
CACHE_TABLE=vibegraph-embedding-cache

# Bedrock Models
CLAUDE_MODEL_ID=anthropic.claude-3-5-sonnet-20241022-v2:0
TITAN_MODEL_ID=amazon.titan-embed-text-v2:0

# Logging
LOG_LEVEL=info
```

## Error Handling

All handlers implement consistent error handling:

- **400 Bad Request**: Invalid input parameters
- **401 Unauthorized**: Missing or invalid authentication
- **404 Not Found**: Resource not found (user, session, etc.)
- **500 Internal Server Error**: Service failures (Claude, Titan, DynamoDB)
- **503 Service Unavailable**: Temporary service issues with retry logic
