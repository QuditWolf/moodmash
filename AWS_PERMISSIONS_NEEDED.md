# AWS Permissions Required

## Current Status

✅ **Backend configured correctly** - Using us-east-1 region  
✅ **Credentials working** - AWS recognizes the user  
❌ **Missing permissions** - User cannot invoke Bedrock models  

## Error Message

```
User: arn:aws:iam::682839842582:user/moodmash is not authorized to perform: 
bedrock:InvokeModel on resource: arn:aws:bedrock:us-east-1::foundation-model/...
because no identity-based policy allows the bedrock:InvokeModel action
```

## What You Need to Do in AWS Console

### Step 1: Add Bedrock Permissions to IAM User

1. Go to **AWS Console** → **IAM** → **Users**
2. Click on user: **`moodmash`**
3. Click **"Add permissions"** button
4. Select **"Attach policies directly"**
5. Search for: **`AmazonBedrockFullAccess`**
6. Check the box next to it
7. Click **"Add permissions"**

### Step 2: Enable Bedrock Models (if not already done)

1. Go to **AWS Console** → **Bedrock** service
2. **IMPORTANT:** Make sure you're in **us-east-1** region (top right corner)
3. Click **"Model access"** in the left sidebar
4. Click **"Manage model access"** or **"Request model access"**
5. Enable these models:
   - ✅ **Claude 3.5 Sonnet** (anthropic.claude-3-5-sonnet-20241022-v2:0)
   - ✅ **Titan Embeddings G1 - Text v2** (amazon.titan-embed-text-v2:0)
   - ✅ **Titan Image Generator** (amazon.titan-image-generator-v1) - Optional for DNA cards
6. Click **"Request model access"**
7. Wait for approval (usually instant)

## Alternative: Minimal Custom Policy

If you don't want to use `AmazonBedrockFullAccess`, you can create a custom policy with just the needed permissions:

1. Go to **IAM** → **Policies** → **Create policy**
2. Click **JSON** tab
3. Paste this:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel",
        "bedrock:InvokeModelWithResponseStream"
      ],
      "Resource": "*"
    }
  ]
}
```

4. Name it: `VibeGraphBedrockAccess`
5. Create policy
6. Go back to **Users** → **moodmash** → **Add permissions**
7. Attach the custom policy

## After Adding Permissions

Once you've added the permissions and enabled the models, run this command to test:

```bash
curl -X POST http://localhost:8000/quiz/section1/start \
  -H "Content-Type: application/json" \
  -d '{}'
```

You should see quiz questions generated by Claude!

## Verification

To verify permissions are working, you can also use AWS CLI:

```bash
aws bedrock list-foundation-models --region us-east-1
```

This should list all available models without errors.
