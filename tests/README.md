# Integration Tests

This directory contains integration tests for VibeGraph that verify end-to-end workflows across services.

## Overview

Integration tests validate complete user flows and inter-service communication:

- **Quiz Flow**: Complete adaptive quiz from Section 1 through completion
- **Matching Flow**: Finding taste matches between users
- **Growth Path Flow**: Generating personalized recommendations
- **Analytics Flow**: Generating behavioral insights

## Prerequisites

Before running integration tests, ensure:

1. All services are running:
   ```bash
   make up
   ```

2. Services are healthy:
   ```bash
   make wait-healthy
   ```

3. Database is initialized:
   ```bash
   make init-db
   ```

## Running Tests

Run all integration tests:
```bash
make test-integration
```

Run specific test file:
```bash
pytest tests/integration/test_quiz_flow.py -v
```

Run specific test:
```bash
pytest tests/integration/test_quiz_flow.py::TestQuizFlow::test_complete_quiz_flow -v
```

## Writing Integration Tests

Integration tests should:

1. **Test complete workflows**: Verify entire user journeys, not isolated functions
2. **Use real HTTP requests**: Call actual API endpoints
3. **Verify data persistence**: Check that data is correctly stored in DynamoDB
4. **Test error scenarios**: Verify proper error handling and recovery
5. **Be independent**: Each test should set up and clean up its own data

### Example Structure

```python
import pytest
import requests

class TestMyFlow:
    """Integration test for my workflow."""
    
    @pytest.fixture
    def api_client(self):
        """Create API client for testing."""
        return requests.Session()
    
    def test_complete_flow(self, api_client):
        """Test complete workflow from start to finish."""
        # Step 1: Initial request
        response = api_client.post(
            "http://localhost:8000/api/endpoint",
            json={"data": "value"}
        )
        assert response.status_code == 200
        
        # Step 2: Verify result
        result = response.json()
        assert "expected_field" in result
```

## Configuration

Integration tests use the following configuration:

- **API Base URL**: `http://localhost:8000`
- **Timeout**: 30 seconds per request
- **Retry Logic**: No automatic retries (tests should be deterministic)

## Debugging Failed Tests

If integration tests fail:

1. **Check service health**:
   ```bash
   make health
   ```

2. **View service logs**:
   ```bash
   make logs
   ```

3. **Check specific service**:
   ```bash
   make logs-backend
   ```

4. **Run test with verbose output**:
   ```bash
   pytest tests/integration/test_quiz_flow.py -v -s
   ```

5. **Check database state**:
   ```bash
   make shell-backend
   python scripts/check-db.py
   ```

## Test Data

Integration tests create test data with predictable IDs:

- User IDs: `test-user-integration`, `test-user-123`, etc.
- Session IDs: Generated by the API
- Embedding IDs: Generated by the API

Test data is isolated and should not interfere with development data.

## CI/CD Integration

Integration tests run in CI/CD pipeline after:
1. Docker images are built
2. Services are started
3. Health checks pass
4. Database is initialized

Tests must pass for deployment to proceed.

## Known Limitations

- **Timing-dependent tests**: Some tests may be sensitive to timing (e.g., session expiration)
- **External dependencies**: Tests require LocalStack for AWS services
- **Data cleanup**: Tests should clean up their data, but manual cleanup may be needed
